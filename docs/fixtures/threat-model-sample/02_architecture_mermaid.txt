--- Trust boundaries (for threat modeling) ---

1. Boundary: Browser ↔ CDN — untrusted client, XSS/CSRF surface on FE.
2. Boundary: FE ↔ API — service JWT, CORS allowlist app.medflow.com.
3. Boundary: API ↔ PostgreSQL — TLS, IAM auth, PHI at rest KMS.
4. Boundary: API ↔ Stripe — card data never touches MedFlow disks (tokenization only).
5. Boundary: API ↔ OpenAI — clinical note snippets may leave PHI zone if summarization enabled; feature-flagged per tenant.
6. Boundary: API ↔ Twilio — signaling/metadata in MedFlow; media encrypted E2E per vendor claim (validate contract).

--- Known deployment facts ---

- Region: us-east-1 primary; no cross-region replication for PHI bucket yet (DR gap).
- Auth: patients use magic-link (email); providers use Okta MFA.
